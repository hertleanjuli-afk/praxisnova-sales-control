# Runbook T3.1: Apollo Emailer Campaigns Bau/Handwerk pausieren

Erstellt: 2026-04-21
Operator: Angie (Live-Apollo-Action), Claude Code (nur Dokumentation, keine Live-Calls)
Teil von: Track 3 Agents ICP-Switch
Scope: Apollo-UI-seitige Emailer-Campaigns. Interne Brevo-Sequenzen laufen separat ueber Migration v11.

## Kontext

Track 3 pivotiert von Bau/Handwerk auf PropTech/Hausverwaltung/Kanzlei/Agentur.
Zwei getrennte Layer sind betroffen:

1. **Interne Brevo-Sequenzen** (lib/sequences/bauunternehmen.ts + handwerk.ts):
   Pause via Migration v11 (sequence_status='active' -> 'paused'). Automatisch
   durch process-sequences cron respektiert (bestehender status-Check).

2. **Apollo Emailer Campaigns** (von Angie in der Apollo-UI angelegt, NICHT via API aus
   diesem Repo erzeugt): Muessen separat in Apollo pausiert werden. Dieses Runbook.

## Warum kein automatisierter Apollo-Call aus Claude Code

- Apollo-Campaign-Pause ist Live-Ops mit sofortigem Effekt auf Kontakte in Touch-Chains.
- CLAUDE.md-Regel: keine Live-DB-Writes und keine Live-Customer-facing Aktionen durch
  Claude Code ohne Angie-Freigabe. Apollo faellt in diese Kategorie.
- Apollo-MCP (claude.ai-Connector) ist in Claude Code CLI nicht authentifiziert.

## Schritt-fuer-Schritt (Angie-Aktionen)

### Vorbereitung (5 Min)

1. Apollo oeffnen: https://app.apollo.io/
2. Navigate: **Engage** -> **Sequences**
3. Filter: **Status = Active**

### Identifikation (10 Min)

4. Suche nach Sequenzen mit Tag oder Name-Pattern:
   - `bau`
   - `handwerk`
   - `fenster-tueren-fassaden`
   - `bau-handwerk`
   - `handwerkskammer`
   - `SHK` (Sanitaer-Heizung-Klima)
5. Fuer jede gefundene Sequenz in Tabelle (nachstehend) notieren:
   - Sequence-Name
   - Sequence-ID (aus URL-Segment `/sequences/{id}`)
   - Anzahl aktiver Kontakte
   - Erstellt-am

### Pause-Aktion (15 Min)

6. Fuer jede Sequenz aus der Liste:
   a. Sequenz oeffnen
   b. Alle aktiven Kontakte aktivieren (Checkbox "Select All Active")
   c. Action: **Remove from sequence** ODER **Pause all**
   d. Reason: "ICP-Pivot 2026-04-21 Bau/Handwerk raus, alte Sequenzen eingefroren"
   e. **NICHT loeschen**. Sequenz bleibt fuer Archiv/Reactivation.
7. Sequenz-Status auf **Archive** setzen, damit sie nicht erneut gestartet werden kann.

### Dokumentation (10 Min)

8. Tabelle unten in diesem Runbook ausfuellen (Sequenz-Name + ID + Kontakte pausiert).
9. Optional: Parallel-Log in DB via SQL:

```sql
INSERT INTO sequence_pause_log (sequence_type, reason, operator, paused_at)
VALUES
  ('apollo-bau-mahnwesen',        'Apollo UI pause ICP-Pivot', 'angie-manual', now()),
  ('apollo-handwerk-angebote',    'Apollo UI pause ICP-Pivot', 'angie-manual', now()),
  ('apollo-fenster-tueren',       'Apollo UI pause ICP-Pivot', 'angie-manual', now());
-- Namen anpassen an echte Apollo-Sequenzen
```

## Rollback

Falls Pivot revidiert wird:

1. Apollo-Sequenz-Status zurueck auf Active setzen
2. Kontakte manuell re-add in Sequenz
3. DB-Log: `DELETE FROM sequence_pause_log WHERE operator='angie-manual' AND reason LIKE '%Apollo UI pause ICP-Pivot%';`
